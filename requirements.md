# 天気情報確認アプリ 要件定義

## 1. 目的
- 学習目的（API使用、`useForm`利用、状態管理、パラメーター付きURL）

## 2. 対象ユーザー
- 自分
- 一般ユーザー

## 3. ユースケース
- 出かける前
- 予定を決める時（2週間以内）

## 4. 提供価値
- モダンUI
- 機能がシンプル（地名検索、天気表示、気温、風）
- 占い付き

## 5. 必須機能（MVP）
- 都市名検索
- 現在天気
- 気温
- 温度
- 天気アイコン
- 風力

## 6. 任意機能（拡張）
- 花粉情報
- 占い情報
- 雨雲レーダー
- 履歴
- お気に入り都市

## 7. 非機能要件
- スマホアプリ対応
- API取得エラー表示
- ローディング表示

## 8. UI/UXの方針
- シンプル
- パステルカラー

## 9. データ/外部API
- Open-Meteo API

## 10. 技術制約
- React（`useForm`, `useQuery`）
- TypeScript
- Tailwind
- Java
- Spring Boot
- SQL
- 無料枠API

## 11. スコープ外
- 通知
- アカウント設定
